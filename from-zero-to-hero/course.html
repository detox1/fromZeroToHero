Modul 1

Mindset developera

Šta je programiranje?

Programiranje je davanje instrukcija računaru na jeziku koji on razumije. Računar radi tačno ono što mu kažeš — ne ono što si mislio da si rekao. U početku je ovo frustrirajuće. Onda počneš cijeniti tu preciznost.

Najvažnija vještina nije pisanje koda. Najvažnija vještina je razlaganje problema na manje dijelove. Svaki feature koji ikad budeš gradio je veliki problem napravljen od malih. Posao developera je razložiti veliko na malo, dok svaki mali dio nije dovoljno jasan da se kodira.

Više vremena ćeš čitati kod nego što ga pišeš

Na prvom danu pravog posla, neko ti da codebase od 100.000 linija. Moraš ga navigirati, razumjeti, i mijenjati ga bez da ga pokvariš. Ovaj kurs uči tu vještinu — ne samo "kako napisati petlju".

Greške nisu neuspjeh — one su informacija

Svaka poruka o grešci je računar koji ti govori tačno šta je pošlo naopako. Tvoj posao je pažljivo je pročitati. Developeri koji napreduju brzo su oni koji čitaju poruke o greškama umjesto da panikuju.

Dva načina rada

Način radaOpis
Flow (tok)Znaš šta gradiš, gradiš to, sve ima smisla
Zapeo siNešto ne radi, ne znaš zašto, ništa ne funkcioniše

Znati kako prijeći iz "zapeo sam" u "flow" — to je prava vještina. Modul 14 to pokriva u detalje.

Modul 3

Komandna linija

Komandna linija (terminal, shell, konzola) je tekstualni interfejs prema računaru. Kucaš komande umjesto klikanja ikonama. Kad se naučiš na ovo, radit ćeš duplo brže.

Osnove navigacije

bash
pwd                  # Gdje sam? (Print Working Directory)
ls                   # Lista fajlova u trenutnoj mapi
ls -la               # Lista fajlova uključujući skrivene
cd ime-mape          # Uđi u mapu
cd ..                # Idi gore jedan nivo
cd ~                 # Idi u home direktorij

mkdir moja-mapa      # Napravi novu mapu
cp fajl.txt kopija.txt  # Kopiraj fajl
mv fajl.txt novo.txt    # Preimenuj/premjesti fajl
rm fajl.txt          # Obriši fajl (pažnja — nema korpa!)
cat fajl.txt         # Ispiši sadržaj fajla
code .               # Otvori VS Code u trenutnoj mapi

Vježba — uradi ovo odmah

  1. Otvori terminal (Ctrl + ` u VS Code)
  2. Ukucaj cd ~ — idi u home mapu
  3. Ukucaj mkdir moj-prvi-projekat — napravi mapu
  4. Ukucaj cd moj-prvi-projekat — uđi u nju
  5. Ukucaj pwd — potvrdi gdje si
  6. Ukucaj code . — otvori VS Code ovdje

Modul 5

Git — Kontrola verzija

Git nije opcija. To je najosnovniji alat u profesionalnom razvoju softvera. Omogućava ti da snemaš snapshote koda, poništiš greške, sarađuješ s drugima i isporučuješ s povjerenjem.

Osnovna ideja — commit: Commit je snimljeni snapshot tvog koda u jednom trenutku. Razmišljaj o tome kao o snimanju igrice. Uvijek se možeš vratiti nazad.

Svakodnevne komande

bash
git init                          # Pokreni novi git repozitorij
git status                        # Koji su fajlovi promijenjeni?
git add ime-fajla.txt             # Dodaj fajl u staging
git add .                         # Dodaj sve promijenjene fajlove
git commit -m "dodaj login feature" # Snimi snapshot sa porukom
git log                           # Vidi historiju commitova

Branches — rad na featuresima sigurno

Branch je paralelna verzija tvog koda. Praviš ga za rad na nekom featureu, a onda ga spojiš kad je završen. Znači da je main kod uvijek stabilan.

bash
git checkout -b feature/dodaj-login  # Napravi i prebaci se na novi branch
git checkout main                    # Vrati se na main
git merge feature/dodaj-login        # Spoji feature u main
git branch -d feature/dodaj-login    # Obriši branch nakon spajanja

Rad sa GitHub-om

bash
git clone https://github.com/user/repo.git  # Preuzmi repozitorij
git push origin main                         # Pošalji svoje commitove
git pull                                      # Preuzmi tuđe commitove
git fetch                                     # Provjeri promjene bez spajanja

Format commit poruke (koristi se u ovom projektu)

text
feat(candidate): dodaj bulk CSV import s validacijom
fix(chatbot): gracefully handle Ollama timeout
docs(api): dodaj ProspectXCandidate endpoint referencu
refactor(auth): simplify token validaciju

Format: tip(scope): kratki opis — ovo se zove Conventional Commits.

Zlatna pravila Gita

  1. Commituj često. Mali commitovi su lakše razumljivi i mogu se poništiti.
  2. Nikad direktno u main na timskom projektu. Uvijek koristi branch.
  3. Piši smislene commit poruke. "fix stuff" je beskorisan za 6 mjeseci.
  4. Pull prije push-a. Preuzmi tuđe promjene prije nego uploaduješ svoje.

Modul 8

Baze podataka i SQL

Skoro svaka aplikacija čuva podatke. Ti podaci žive u bazi podataka. SQL (Structured Query Language) je jezik kojim razgovaraš s njom.

Šta je relaciona baza podataka?

Tabela: Kandidati
IdImeKomId
1Emir10
2Sara11
Tabela: Kompanije
IdNaziv
10Acme d.o.o.
11Globex d.d.

KomId (KompanijaId) je strani ključ — povezuje se na tabelu Kompanije

Osnovni SQL — četiri operacije (CRUD)

sql
-- CREATE — umetni nove podatke
INSERT INTO Kandidati (Ime, Prezime, Email)
VALUES ('Emir', 'Hodžić', 'emir@example.com');

-- READ — upitaj podatke
SELECT Id, Ime, Prezime, Email
FROM Kandidati
WHERE Aktivan = 1
ORDER BY Prezime ASC;

-- UPDATE — promijeni postojeće podatke
UPDATE Kandidati
SET Email = 'novi@example.com'
WHERE Id = 42;

-- DELETE — obriši podatke
DELETE FROM Kandidati
WHERE Id = 42;

Spajanje tabela (JOIN)

sql
-- Dohvati ime kandidata I naziv njihove kompanije
SELECT
    k.Ime,
    k.Prezime,
    kom.Naziv AS NazivKompanije
FROM Kandidati k
INNER JOIN Kompanije kom ON k.KomId = kom.Id
WHERE k.Aktivan = 1;

ORM — Entity Framework Core

Pisanje raw SQL-a za svaku operaciju je zamorno i sklono greškama. ORM mapira C# klase na tabele baze automatski.

Pisanje kontrolera

csharp
[ApiController]
[Route("api/[controller]")]
public class KandidatiController : ControllerBase
{
    private readonly CandidateFeature _feature;

    public KandidatiController(CandidateFeature feature)
    {
        _feature = feature;
    }

    [HttpGet("{id}")]
    [Authorize(Roles = "Administrator,Manager,Recruiter")]
    public async Task<IActionResult> GetById(int id)
    {
        var kandidat = await _feature.GetByIdAsync(id);
        if (kandidat == null) return NotFound();
        return Ok(kandidat);
    }
}
Primijeti: kontroler je tanak. On samo rukuje HTTP pitanjima (status kodovi, routing, auth). Sva poslovna logika je u Feature klasi.

Modul 11

Arhitekturni paterni

Ovo su ponavljajuća rješenja na uobičajene probleme. Vidjet ćeš ih svugdje, uključujući ovaj projekat. Ne trebaš ih memorisati na dan 1 — vrati se ovdje kad ih sretneš.

🏗️
Clean Architecture
4 sloja sa strogim pravilima. Prezentacija → Aplikacija → Domena → Infrastruktura. Svaki sloj može samo zavisiti od sloja desno od njega.
📦
Repository Pattern
Klasa koja rukuje svim pristupom bazi za jedan tip entiteta. Tvoj feature kod ne zna jesu li podaci u SQL Serveru, PostgreSQL-u ili fajlu.
⚡
CQRS
Command Query Responsibility Segregation — razdvoji čitanje od pisanja. Dapper za čitanja, EF Core za pisanja. Svaka strana je optimizovana za svoju svrhu.
💉
Dependency Injection
Ne pravi zavisnosti unutar klase — primi ih. Framework ih ubrizgava. Zamjena implementacije je jedna linija koda. Testiranje je lako.
📡
Observer / Pub-Sub
Kad se desi nešto (kandidat je napravljen), objavi događaj. Pretplatnici reaguju nezavisno. Ovaj projekat koristi MassTransit + RabbitMQ.
🎯
Feature Pattern
Umjesto dijeljenja koda po tipu (svi kontroleri zajedno), ovaj projekat dijeli po featureu. Svih 28 operacija kandidata žive u CandidateFeature.
Pravilo 80/20 arhitekture: Ne trebaš savršenu arhitekturu od prvog dana. Počni s najjednostavnijim što funkcioniše. Refaktoriraj kad osjetiš bol. Paterni gore postoje jer su developeri osjetili bol bez njih. Razumijevanje ZAŠTO postoje važnije je od mehaničke primjene.

AI kao tvoj asistent

Imaš Claude Code ugrađen u editor. Evo kako ga koristiti da budeš 10x produktivniji bez da postaješ ovisan.

AI je dobar za
  • Objašnjavanje koda
  • Pisanje boilerplate-a
  • Pronalazak bugova
  • Prijedloge poboljšanja
  • Debugging greški
Ti moraš raditi ovo
  • Razumjeti zahtjeve
  • Procjenjivati arhitekturu
  • Testirati da li zaista radi
  • Debugging u produkciji

Pravi workflow s AI-om

  1. Ti razumiješ problem. Čitaj kod, grešku, zahtjev.
  2. Ti formuiraš hipotezu. "Mislim da ovo ne radi jer..."
  3. Pitaš AI da pomogne izvršiti. "Možeš li napisati SQL upit za ovo?"
  4. Ti pregledaš šta AI vrati. Čitaj svaku liniju. Pitaj "zašto" ako nešto nije jasno.
  5. Ti testiraš. Pokreni to. Vidi radi li. Provjeri rubne slučajeve.

Primjeri prompta

Loš prompt: "Popravi mi kod"
Dobar prompt: "U crm-backend/Application/Features/CandidateFeature.cs, metoda SearchAsync vraća nula rezultata kad tražim 'emir'. Pretraga ide kroz Database.Dapper/Repositories/GetCandidatesBasedOnSearch.cs. Možeš li proći kroz kod i reći zašto bi ovo vraćalo prazne rezultate?"

Modul 15

Kako dobiti posao

Šta kompanije zapravo žele od junior developera

1
Najvažnije
Možeš li učiti? Više od bilo koje specifične tehnologije.
2
Komunikacija
Možeš li objasniti šta si gradio i zašto?
3
Problem solving
Odustaješ ili istražuješ metodički?
4
Kvalitet
Pišeš čitljiv kod koji kolega može pročitati za 6 mjeseci?

Checklist za portfolio projekat

  • Jasna svrha (rješava pravi problem)
  • README s opisom, uputama za pokretanje i screenshotima
  • Čist, čitljiv kod
  • Backend API i frontend koji ga poziva
  • Baza podataka s nekim zanimljivim upitima
  • Deploy negdje (Azure free tier, Railway, Render)
  • Testovi za osnovnu poslovnu logiku

Kako pričati o ovom projektu na intervjuu

Radio si na pravom enterprise sistemu: CRM s AI asistentom, izgrađen u .NET Clean Architecture s Vue frontendom, SQL Server bazom i chatbotom pogonjenim lokalnim LLM-om. To nije tutorial projekat — to je pravi produkcijski sistem.

Vježbaj objašnjavati:
• Šta sistem radi (ne-tehnički, kao da pričaš s product managerom)
• Na čemu specifično si radio (budi iskren i precizan)
• Problem koji si riješio — bug koji si popravio, feature koji si napravio
• Šta bi uradio drugačije — pokazuje da možeš reflektovati i učiti

Tvoj prvi posao

Tvoj prvi posao neće biti savršen posao. Bit će mjesto gdje učiš brže nego što bi ikad mogao sam, jer si okružen ljudima koji su već napravili greške koje ti tek trebaš napraviti.

Traži firmu gdje
  • Možeš vidjeti pravi kod prije prihvatanja
  • Developeri odgovaraju na pitanja
  • Code review je pravi proces
  • Tim koristi version control ispravno
Izbjegavaj firme gdje
  • Svi su juniori
  • Niko ne radi code review
  • "Refaktorisat ćemo to kasnije" je stalni odgovor
  • Nema version controla
