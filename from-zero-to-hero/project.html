Zamislite: Fabrika igračaka 🏭

Zamoislite da imate fabriku igračaka. Fabrika ima različite sobe i svaka soba ima jedan posao. Niko ne radi posao drugog. To je cijela ideja ovog projekta.

Fabrika ima dvije zgrade:
JobPortal/ — stara zgrada (zamjenjuje se, ignoriši je)
src/ — nova zgrada (ovo je prava, organizovana)

Nova zgrada ima 4 sprata

Sprat 4 — ULAZNA VRATA
Hosts/ — API + Website
JobPortal.Api — .NET vrata · Skipsearch.Client — website koji korisnici vide
Sprat 3 — SOBA S ALATIMA
Infrastructure/ — Baza, Servisi, Email
AppDbContext, Repositories, TokenService, SendEmailService, LinkedIn adapter
Sprat 2 — MOZAK
Application/ — Logika, Handleri, CQRS
Commands, Queries, Handlers — čista logika, nema baza nema weba
Sprat 1 — PRAVILA (SRCE)
Domain/ — Entiteti, Interfejsi
User, AppliedJob, LikedJob — blueprinti. IUserRepository — obećanja. Nula vanjskih zavisnosti.
Zlatno pravilo: Svaki sprat razgovara samo sa spratom ISPOD sebe. Nikad ne preskačeš spratove. Nikad ne ideš GORE.

Sprat 2

Application — Mozak 🧠

Ovdje se dešava razmišljanje. Nema koda za bazu. Nema web koda. Samo čista logika.

Koristi CQRS — modernu riječ za razdvajanje stvari u dva segmenta:

Queries = samo ČITANJE
  • GetJobsQuery — daj mi listu poslova
  • GetJobDetailsQuery — daj mi detalje posla #5
  • GetUserQuery — daj mi info korisnika
Nikad ne mijenja podatke. Samo čita.
Commands = MIJENJANJE
  • CreateUserCommand — napravi korisnika
  • ApplyForJobCommand — apliciraj na posao
  • LikeJobCommand — lajkaj posao
Mijenja podatke. Vraća uspjeh/neuspjeh.

Svaki Query/Command ima Handler — radnika

csharp
// Pitanje: "daj mi listu poslova"
public class GetJobsQuery : IQuery<List<JobDto>>
{
    public string? SearchTerm { get; set; }
}

// Odgovor: handler koji radi posao
public class GetJobsQueryHandler : IQueryHandler<GetJobsQuery, List<JobDto>>
{
    private readonly IJobExternalService _jobService;  // obećanje iz Domain sloja

    public GetJobsQueryHandler(IJobExternalService jobService)
    {
        _jobService = jobService; // ubrizgano izvana — handler ne pravi ovo sam
    }

    public async Task<List<JobDto>> Handle(GetJobsQuery query, CancellationToken ct)
    {
        return await _jobService.GetJobsAsync(query.SearchTerm);
    }
}
Handler prima IJobExternalService — interfejs (obećanje). Ne zna i nije mu bitno da je JobService u Infrastructure sloju prava klasa koja pravi HTTP pozive.

Ispunjavanje obećanja — Repository implementacija

csharpInfrastructure/Repositories/UserRepository.cs
public class UserRepository : IUserRepository  // ispunjava obećanje!
{
    private readonly AppDbContext _context;

    public UserRepository(AppDbContext context)
    {
        _context = context;
    }

    public async Task<User> GetByIdAsync(Guid id)
    {
        return await _context.Users.FindAsync(id); // stvarni SQL upit via EF Core
    }

    public async Task AddAsync(User user)
    {
        await _context.Users.AddAsync(user); // priprema INSERT
    }
}

Soba za žičanje — DependencyInjection.cs

csharpInfrastructure/DependencyInjection.cs
services.AddScoped<IUserRepository, UserRepository>();
// "Kad iko pita za IUserRepository, daj mu UserRepository"

services.AddScoped<IUnitOfWork, UnitOfWork>();
services.AddScoped<ISendEmailService, SendEmailService>();
services.AddScoped<IJobExternalService, JobService>();
Ovo je magija: Mozak (Application) kaže "daj mi IUserRepository". Soba za žičanje kaže "evo ti UserRepository". Mozak ne zna i nije mu bitno šta točno dobija — samo da ispunjava obećanje.

Paterni

Repository + Unit of Work

Repository Pattern — Radnik u skladištu

Problem: Ako pišeš SELECT * FROM Users direktno u poslovnoj logici, zauvijek si vezan za SQL Server. Ne možeš testirati. Ne možeš zamijeniti bazu.

Rješenje: Napravi "posrednika" (repository) čiji jedini posao je razgovor s bazom.

→
Poslovna logika
Govori: "Daj mi sve korisnike"
→
IUserRepository
Interfejs — posrednik
→
UserRepository → SQL Server
Pravi SQL upit — poslovna logika to ne vidi
Ako zamijenite SQL Server za MongoDB sutra — samo prepisujete UserRepository. Poslovna logika se uopće ne mijenja.

Auth

JWT Autentifikacija 🔐

Kad se prijaviš, server te ne pamti između zahtjeva (HTTP je bez stanja). Umjesto toga, daje ti token — potpisani string koji dokazuje ko si.

📝
1. POST /api/auth/login s emailom i lozinkom
🔍
2. Server provjerava kredencijale u bazi
🎫
3. Server pravi JWT token: eyJhbGciOi... (enkodira ID korisnika, role, istek, potpis)
💾
4. Klijent čuva token (localStorage ili cookie)
📤
5. Svaki budući zahtjev uključuje token: Authorization: Bearer eyJ...
✅
6. Server validira potpis na svakom zahtjevu — validan → autenticiran, istekao/izmijenjen → 401
csharpInfrastructure/Services/TokenService.cs
// TokenService.cs u Infrastructure sloju pravi ove tokene
public string CreateToken(User user)
{
    var claims = new[]
    {
        new Claim(ClaimTypes.NameIdentifier, user.Id.ToString()),
        new Claim(ClaimTypes.Email, user.Email),
    };

    var key = new SymmetricSecurityKey(Encoding.UTF8.GetBytes(_secret));
    var token = new JwtSecurityToken(
        claims: claims,
        expires: DateTime.UtcNow.AddHours(24),
        signingCredentials: new SigningCredentials(key, SecurityAlgorithms.HmacSha256)
    );

    return new JwtSecurityTokenHandler().WriteToken(token);
}

export const useAuthStore = defineStore('auth', {
    state: () => ({
        user: null as User | null,
        token: null as string | null,
        isAuthenticated: false
    }),
    actions: {
        async login(email: string, password: string) {
            const { token, user } = await $fetch('/api/auth/login', {
                method: 'POST',
                body: { email, password }
            })
            this.token = token
            this.user = user
            this.isAuthenticated = true
        }
    }
})

Composables — ponovna upotreba logike

typescriptapp/composables/useJobs.ts
// "Kako dohvatiti poslove?" — koristi bilo koja stranica koja treba poslove
export function useJobs() {
    const jobs = ref([])
    const isLoading = ref(false)

    async function fetchJobs(searchTerm?: string) {
        isLoading.value = true
        jobs.value = await $fetch('/api/jobs', { params: { search: searchTerm } })
        isLoading.value = false
    }

    return { jobs, isLoading, fetchJobs }
}
